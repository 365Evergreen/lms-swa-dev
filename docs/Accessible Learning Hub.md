# Accessible Learning Hub

An Azure SWA learning management system,

## Purpose of this plan

This plan serves as a blueprint for addressing the business problem described above, outlining the user requirements, data model, and technology that will be used to get things done.

## Personas

**1. Learner**

Individual who browses, enrolls, completes courses, and tracks progress.

**2. Course administrator**

Person who manages courses, modules, and assessments.

**3. Business owner**

Person responsible for the overall success, strategy, and oversight of the learning
management app.

**4. Course teacher**

Person who delivers course material, engages with learners, and assesses module
completion.

## User stories

### Learner

As a learner, I want to:



- Browse available courses so that I can find topics of interest

- Register for courses using forms so that I can start learning new subjects

- View and track my progress in modules so that I know how much I have
  completed.

- Complete modules and assessments so that I can gain new knowledge and
  verify my understanding.

- Switch between dark and light modes so that I can reduce eye strain
  according to my preference.

- Navigate the app with a collapsible sidebar so that I can focus on content or
  multitask more efficiently.

- Browse and update my profile so that my learning achievements are accurate.

- Showcase my achievements so that I can celebrate and share my milestones.

- Use keyboard navigation and screen reader support so that the app is
  accessible to everyone.
  
  ---
  
  ### Content administrator
  
  As a content administrator, I want to:

- organise modules and assessments within courses so that learning journeys
  are structured.

- View learner progress and achievements so that I can monitor engagement
  and outcomes.

- Ensure app compliance with accessibility standards so that all users have an
  inclusive experience.

- Update course themes and customise UI tokens so that the app maintains a
  consistent brand and usability.
  
  ---
  
  ### Business owner
  
  As a buisness owner, I want to:
  
  Monitor overall course and learner engagement so that I can measure the
  app's effectiveness
  
  Review analytics and reports on course registration, completion, and user
  achievements so that I can guide business decisions.
  • Set strategic goals for the learning platform so that the organisation
  continues to meet educational needs.
  • Oversee compliance with accessibility standards so that the app serves all
  stakeholders appropriately.
  
  ---
  
  ### Content editor
  
  As a content editor, I need to:

- Create and update course content so that learners have accurate and relevant
  material.

- --Organise materials into modules and assessments so that learning is

- Collaborate with course administrators and teachers so that the course
  curriculum remains coherent.

- Ensure course content supports different themes and accessibility features so
  that all users can engage.
  
  ---
  
  ### Course teacher
  
  As a course teacher, I want to:
  

- Deliver and explain module content so that learners understand the material
  thoroughly.

- Assess learner submissions and provide feedback so that learners improve
  their knowledge.

- Track learner progress in assigned courses so that support can be offered
  where needed.

- Communicate with learners about course questions and achievements so that
  engagement is maintained.
  
  
  ## User processes
  
  The processes for solving the business problem. Each process appears as a card next to this document.
  
  
  
  
  ### Learner Course Engagement
  
  This process covers the end-to-end learner experience: browsing courses, registering, completing modules and assessments, tracking progress, managing profiles, and showcasing achievements.
  
   It also includes accessibility and UI customisation features to
  ensure an inclusive and user-friendly experience.
  
  
  ### Course and Content Management
  
  This process involves the creation, organisation, and maintenance of courses, modules, and assessments. It includes collaboration between course administrators, content editors, and teachers to ensure content is accurate, structured, and accessible.
  
  
  
  
  ### Assessment and Feedback
  
  This process focuses on the delivery of module content, assessment of learner
  submissions, and provision of feedback. 
  
  Teachers track learner progress and communicate with learners to support their educational journey.
  
  
  
  
  ### Platform Accessibility and Theming
  
  This process ensures the app complies with accessibility standards and provides UI
  customisation options such as dark/light modes and collapsible sidebar. 
  
  It involves administrators, editors, and business owners overseeing accessibility and theming for a consistent, inclusive experience.
  
  
  
  
  ### Business Oversight and Analytics
  
  This process covers strategic oversight, monitoring engagement, reviewing analytics, and setting goals for the learning platform. 
  
  Business owners ensure the platform meets organisational objectives and serves all stakeholders.

## UI/design

The app will:

- use Fluent UI components, themes and tokens

- have a collapsible sidebar, dark and light modes

- comply with common accessibility standards.
